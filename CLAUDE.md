## Início obrigatório de cada sessão

1. Leia o `MEMORY.md` na raiz do projeto
2. Leia `CHANGELOG.md`, `BUGS.md`, `ROADMAP.md` se relevantes à tarefa
3. Identifique o estado atual e informe o plano antes de agir

---

## Regras obrigatórias

- NUNCA remover código funcional
- NUNCA apagar arquivos sem necessidade
- NUNCA quebrar o build
- NUNCA fazer mudanças desnecessárias além do solicitado
- SEMPRE fazer alterações cirúrgicas e mínimas
- SEMPRE validar com `npm run build` antes de finalizar
- SEMPRE preservar o design Academic Dark (`#0f1117` / `#f59e0b`)

---

## Fluxo obrigatório ao final de cada sessão

1. Atualizar `MEMORY.md` (estado atual, histórico, próximo passo)
2. Atualizar `CHANGELOG.md` com as mudanças realizadas
3. Atualizar `BUGS.md` se corrigiu ou encontrou bugs
4. Atualizar `PROMPTS.md` se usou prompts relevantes
5. Atualizar `LINKS.md` se adicionou novos recursos
6. Atualizar `docs/ROADMAP.md` com status das fases
7. Rodar `npm run build` — só concluir com 0 erros
8. `git add .` → `git commit -m "descrição"` → `git push origin main`
9. Informar que o deploy automático foi disparado na Vercel
