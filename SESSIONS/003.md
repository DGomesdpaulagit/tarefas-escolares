# Sessão 003 — Correção de tela preta (variáveis de ambiente Vercel)

**Data:** 2026-05-19
**Etapa:** 1
**Tags:** #bugfix #vercel #env #producao

---

## O que foi feito

- Identificado que variáveis passadas via `--env` no CLI Vercel não foram persistidas no projeto
- Usado Vercel API REST para salvar `VITE_SUPABASE_URL` e `VITE_SUPABASE_ANON_KEY` permanentemente
- Variáveis salvas para os três targets: `production`, `preview`, `development`
- Novo deploy disparado — app voltou a funcionar em produção

---

## Problemas resolvidos

- **BUG-010:** Tela preta em produção — variáveis `VITE_*` não estavam salvas no projeto Vercel
- Descoberto que o Vite injeta variáveis em **build time**, então precisam estar no Vercel antes do build

---

## Lição aprendida

> As variáveis `VITE_*` são injetadas pelo Vite em BUILD TIME.
> Se não estiverem salvas no Vercel **antes** do build, o app falha com tela preta.
