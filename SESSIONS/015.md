## Bugs corrigidos

### Bug 1 — Importação de planilha retornando erro 400
- **Causa 1:** valores de `status`/`priority` da planilha não batiam com os enums do Supabase
- **Causa 2:** datas do Excel chegavam como número serial (ex: `45672`) em vez de string
- **Fix:** adicionados `sanitizeStatus()`, `sanitizePrioridade()` e `parseExcelDate()` em `parseExcel.ts`
- **Fix:** mensagem de erro no modal agora exibe o erro real do Supabase
- **Arquivos:** `client/src/lib/parseExcel.ts`, `client/src/components/ImportarPlanilhaModal.tsx`
- **Commit:** `4e32f37`

### Bug 2 — Avatar upload falhando com 406 / erro ao processar imagem
- **Causa raiz:** bucket `avatars` não existia no Supabase Storage
- **Causa raiz 2:** perfil do usuário não tinha linha na tabela `profiles` (conta criada antes do trigger)
- **Fix (Storage):** removida dependência do Supabase Storage — imagem comprimida via Canvas (256×256 JPEG 82%) e salva como base64 em `profiles.avatar_url`
- **Fix (banco):** `profileService.get()` trocado de `.single()` para `.maybeSingle()`
- **Fix (banco):** `profileService.update()` trocado para `upsert` internamente
- **Fix (componente):** `handleAvatarChange`, `salvarPerfil` e `AbaTema` trocados de `profileService.update()` para `profileService.upsert()`
- **Fix definitivo:** criada linha do perfil diretamente via SQL no Supabase Dashboard
- **Arquivos:** `client/src/services/profileService.ts`, `client/src/services/settingsService.ts`, `client/src/pages/Configuracoes.tsx`
- **Commits:** `725c09a`, `68c446c`, `b50dc3b`, `bdd49d0`

### Bug 3 — notification_settings retornando 409 (conflito)
- **Causa:** upsert sem `onConflict` tentava INSERT quando já havia registro para o usuário
- **Fix:** adicionado `{ onConflict: "user_id" }` no `settingsService.upsertNotifications()`
- **Arquivo:** `client/src/services/settingsService.ts`
- **Commit:** `725c09a`
